What it does

Quake Code is designed to feel fast, focused, and practical for daily coding work. It keeps the UI clean, the workflow inside the terminal, and the defaults tuned for real projects.

Interactive terminal UI

Work in a focused coding interface with sessions, resume, history, model selection, and settings.

Real repo tools

Read files, edit code, search the workspace, and run shell commands without switching context.

Project memory

Use `AGENTS.md`, skills, prompts, themes, and extensions to keep Quake aligned with your workflow.

sessions resume file editing shell tools themes extensions skills

Install

Get Quake Code globally with npm, then launch it from the terminal with `quake` or `quake-code`.

npm install -g @mrquake/quakecode-cli
quake

Command showcase

A few commands that define the everyday Quake workflow.

Start

quake

Open Quake Code and start working in the terminal immediately.

Update

quake update

Pull the latest published Quake version with one clear command.

Project Setup

/init

Create an AGENTS.md starter file for the current project.

Provider Login

/login

Authenticate with a provider and start using Quake with your chosen setup.

Model Switch

/model

Change the active model without leaving the current session.

Resume

/resume

Jump back into an existing session and continue where you left off.

Why teams keep it simple

Quake Code stays intentionally clean: a fast startup, a terminal-first interface, and commands that feel obvious enough to remember.

`quake update`

One obvious update command instead of asking users to remember long npm install strings.

`/init`

Generate an `AGENTS.md` file for the current project and establish clear repo instructions for Quake.

`/resume`

Jump back into previous sessions quickly without losing the flow of longer coding work.

Quake Code — terminal-first coding assistance, designed to stay out of your way.
